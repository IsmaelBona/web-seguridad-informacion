Adquisición y Procesamiento de Datos

Práctica 5: Web Scraping e Integración LLM

1. Objetivo

El propósito de esta práctica es automatizar la extracción masiva de datos no estructurados desde dominios web (diario deportivo). La arquitectura desarrollada permite recolectar texto estructurado, hipervínculos, metadatos de contacto, imágenes y archivos. Esta ingesta de datos es el paso fundamental para la creación de datasets orientados al entrenamiento de modelos de Inteligencia Artificial.

2. Script Python: Extractor Multimodal

El script utiliza requests y BeautifulSoup para recorrer el DOM, descargando los assets locales y estructurando el texto. La información se inyecta como contexto en un modelo GPT-4o-mini vía API para su sintetización. 

import os
from urllib.parse import urljoin
from dotenv import load_dotenv
from openai import OpenAI
import requests
from bs4 import BeautifulSoup

load_dotenv()
API_KEY = os.getenv("OPENAI_API_KEY")
URL = "https://marca.com"
headers = {"User-Agent": "Mozilla/5.0"}

if not os.path.exists("imagenes"):
    os.makedirs("imagenes")

def obtener_noticias(consulta_usuario: str):
    noticias_acumuladas = []
    enlaces_totales = set()
    contactos = set()

    try:
        response = requests.get(URL, headers=headers, timeout=10)
        response.raise_for_status()
        soup = BeautifulSoup(response.text, "html.parser")

        for a_tag in soup.find_all("a", href=True):
            enlaces_totales.add(a_tag["href"])
            if "mailto:" in a_tag["href"]:
                contactos.add(a_tag["href"].replace("mailto:", ""))

        with open("hipervinculos.txt", "w") as f:
            f.write("\n".join(enlaces_totales))

        links = soup.find_all("a", class_="ue-c-cover-content__link")

        for link in links[:20]:
            href = link.get("href")
            if not href: continue
            if href.startswith("/"): href = URL.rstrip('/') + href

            try:
                r = requests.get(href, headers=headers, timeout=8)
                soup2 = BeautifulSoup(r.text, "html.parser")
                h1 = soup2.find("h1", class_="ue-c-article__headline")

                if h1:
                    titular = h1.text.strip()
                    noticias_acumuladas.append(f"TITULAR: {titular}\n---")
            except Exception:
                continue

        return "\n".join(noticias_acumuladas[:8])
    except Exception as e:
        return f"Error: {e}"

client = OpenAI(api_key=API_KEY)
consulta = input("¿Qué quieres saber?: ").strip()
info_noticias = obtener_noticias(consulta)

prompt = f'El usuario pregunta: "{consulta}"\nNoticias:\n{info_noticias}'
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "Resume en 2 frases."},
        {"role": "user", "content": prompt}
    ]
)
print(response.choices[0].message.content)

3. Análisis Teórico y Arquitectura OSINT

a. ¿Qué se puede conseguir con este tipo de programas?

La capacidad de estructurar la web permite habilitar pipelines de datos a gran escala. A nivel de Data Science, es el mecanismo principal para generar corpus de texto orientados al fine-tuning de modelos generativos.

b. OSINT: Recolección masiva sobre entidades

Permite indexar repositorios de código corporativo en busca de credenciales expuestas y extraer correos electrónicos enterrados en metadatos o archivos PDF olvidados en subdominios no asegurados.

c. Limitaciones y problemática en redes sociales

Su viabilidad técnica se desploma ante ecosistemas cerrados. Los obstáculos incluyen el renderizado dinámico mediante JavaScript, bloqueos algorítmicos mediante heurística de IP y sistemas CAPTCHA.

Valoración de la práctica

Dificultad: Media-Baja. El verdadero reto para los ingenieros de inteligencia artificial consiste en programar rutinas de limpieza (Data Cleaning) robustas que soporten las alteraciones constantes en el código fuente de los objetivos web.
