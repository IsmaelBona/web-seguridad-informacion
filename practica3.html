Solución: Cómo arreglarlo

La vulnerabilidad en ambos formularios radica en el uso de la propiedad innerHTML. Esta propiedad toma la cadena de texto introducida por el usuario y fuerza al analizador del navegador a interpretarla como marcado HTML ejecutable.

Para mitigar este fallo y asegurar la aplicación, es imperativo utilizar propiedades que traten la entrada estrictamente como texto plano, neutralizando cualquier etiqueta o script malicioso.

Implementación de código seguro

function procesarBusquedaSegura() {
    var entrada = document.getElementById("campoBusqueda").value;
    document.getElementById("resultadoBusqueda").textContent = entrada;
}

function publicarEstadoSeguro() {
    var estado = document.getElementById("campoEstado").value;
    var elementoMuro = document.getElementById("muroEstado");
    elementoMuro.textContent = "Estado actual: " + estado;
}
