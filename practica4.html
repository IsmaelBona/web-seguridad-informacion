Práctica 4: Romper el HASH - Fuerza Bruta

1. Objetivo

El objetivo es descubrir la contraseña original a partir de un hash SHA-256 proporcionado en un archivo de texto. Para ello, hemos desarrollado scripts en Python que generan combinaciones de caracteres, calculan su hash y los comparan con el objetivo hasta encontrar la coincidencia.

2. Script: Cracker de 4 caracteres

Este script busca combinaciones de letras minúsculas (a-z) con una longitud máxima de 4 caracteres. 

import string, hashlib, itertools

with open("hash_guardado.txt", "r", encoding="utf-8") as archivo:
    hash_objetivo = archivo.readline().strip()

caracteres = string.ascii_lowercase
encontrado = False

for i in range(1, 5):
    if encontrado: break
    for combinacion in itertools.product(caracteres, repeat=i):
        palabra = ''.join(combinacion)
        hash_resultado = hashlib.sha256(palabra.encode()).hexdigest()
        if hash_resultado == hash_objetivo:
            print(f"La combinación es: {palabra}")
            encontrado = True
            break

Resultado obtenido:

vini

3. Script Extra: Contraseña Compleja

Para el desafío extra, incluimos números en el juego de caracteres y aumentamos la longitud de búsqueda inicial a 7 caracteres. 

import string, hashlib, itertools

with open("hash_guardado_extra.txt", "r", encoding="utf-8") as archivo:
    hash_objetivo = archivo.readline().strip()

caracteres = string.ascii_lowercase + string.digits
encontrado = False
num_car = 7

while not encontrado:
    for combinacion in itertools.product(caracteres, repeat=num_car):
        palabra = ''.join(combinacion)
        hash_resultado = hashlib.sha256(palabra.encode()).hexdigest()
        if hash_resultado == hash_objetivo:
            print(f"La combinación es: {palabra}")
            encontrado = True
            break
    num_car += 1

Resultado extra obtenido:

cr7goat

Valoración de la práctica

La dificultad de esta práctica se valora como adecuada para comprender el concepto de coste computacional. Mientras que la clave "vini" se obtiene casi instantáneamente, añadir solo 3 caracteres más y números (como en "cr7goat") aumenta exponencialmente el tiempo de proceso.

Conclusiones técnicas:
  • ✔ La fuerza bruta es efectiva pero ineficiente ante claves largas.
  • ✔ El uso de itertools.product optimiza la generación de memoria.
  • ✔ Se recomienda el uso de "salts" para evitar ataques de diccionario precalculados.
